
# Proyecto Web: Desinformación sobre Seguridad Digital
Este proyecto tiene como objetivo informar sobre la desinformación relacionada con la seguridad digital.

## Páginas del sitio
- index.html (inicio)
- amenazas.html (amenazas digitales)
- consejos.html (recomendaciones)
- contacto.html (contacto)

## Organización del trabajo
Cada integrante trabajó en su propia rama para evitar conflictos y después se integraron los cambios a la rama main mediante pull request.


## Problemática
En la actualidad, la comunidad se enfrenta a un aumento constante de ataques cibernéticos y noticias falsas. Muchos usuarios carecen de los conocimientos básicos para identificar estafas como el phishing o para verificar la veracidad de la información que consumen en redes sociales, lo que pone en riesgo su privacidad y seguridad financiera.

## Objetivo
Desarrollar una plataforma educativa e informativa que brinde herramientas prácticas, consejos de seguridad y guías claras para que los usuarios aprendan a identificar amenazas digitales y adopten mejores prácticas de higiene digital.

## Conflicto generado
El conflicto ocurrió de manera intencional cuando dos integrantes del equipo modificaron simultáneamente el mismo párrafo de descripción en el archivo index.html desde diferentes ramas. Al intentar unir ambas versiones en la rama principal (main), Git detectó cambios contradictorios en las mismas líneas de código.

## Cómo lo resolvimos
Identificación: Localizamos las marcas de conflicto (<<<<<<<, =======, >>>>>>>) dentro del código.
Análisis: El equipo se reunió para comparar ambas propuestas de texto.
Integración: Decidimos crear una versión combinada que aprovechaba lo mejor de ambas redacciones para que el mensaje fuera más claro.
Limpieza: Eliminamos manualmente las marcas de Git, guardamos el archivo y realizamos un nuevo commit final para confirmar la solución.

## Qué aprendimos
Aprendimos que el trabajo colaborativo requiere una comunicación constante para evitar modificar los mismos archivos al mismo tiempo. También reforzamos que los conflictos en Git no son errores fatales, sino herramientas que nos permiten elegir la mejor versión del código de manera democrática y organizada
